4.1
System Overview & Architecture

What is Salon Sanctuary?

Salon Sanctuary is a digital management system designed for beauty salons. It eliminates the traditional paper-based manual record keeping, replacing it with a modern, powerful web application.

The system supports multiple user roles including Admin, Cashier, and Staff — each with their own set of permissions and dashboard views.

Database Design

Database Tables

The database consists of 5 tables as follows:

Table Key Columns Purpose
bookings id, name, phone, service, booking_date, start_time, amount, status Appointments & Walk-ins
services id, name, price, category, image_url, duration All services (Hair, Nails, POS…)
staff id, name, role, phone, email, status Staff members & their roles
customers id, name, phone, email, total_visits, total_spent Registered clients
expenses id, title, category, amount, date Salon operating expenses

🔗 Table Relationships

bookings.customer_id → references the user or admin who created the booking

bookings.service_id → references services.id

services.category → classifies services as: hair, nails, makeup, henna, Dress, Product, POS

4.4
Key Features Implemented
📅

Appointment Management

Admin can book appointments for clients. Each time slot is limited to 3 concurrent bookings. A three-part full name validation is enforced before saving.

🚶

Walk-in System

Clients who arrive without a prior booking can be served immediately. The cashier adds a walk-in record quickly without requiring an appointment.

🛒

Point of Sale (POS)

The cashier can add services to the cart and generate a printable receipt automatically upon checkout.

💰

Finance & Analytics

Revenue and expenses are displayed with interactive charts. Data can be filtered by day, week, or month for detailed financial analysis.

👗

Bridal Rentals

Bridal dress rentals are managed in a dedicated section, with images and individual pricing for each dress item.

👥

Role-Based Access

Admin, Cashier, and Staff each see a different dashboard. Admin accesses all sections; Cashier accesses POS & Walk-in; Staff sees only their assigned bookings.

🔔

Real-time Notifications

When a new booking is added to the system, all connected admin screens immediately receive a toast notification — without any page reload.

📦

Product Inventory

Sellable products appear in the Products section. When an item is sold via POS, the stock count automatically decrements in real time.

🌐

Dual Language (EN / SO)

The system supports both English and Somali languages. The user's language preference is saved in localStorage and persists across sessions.

⏱ Real-time Code (Hub of the System)

The following is the actual source code for the real-time subscription — one of the most critical parts of the system's architecture:

supabase
  .channel('bookings-realtime')
  .on('postgres_changes', {
      event: 'INSERT',
      schema: 'public',
      table: 'bookings'
    }, (payload) => {
      setBookings(prev => [payload.new, ...prev]);
      toast.success("🔔 New booking received!");
    }
  )
  .subscribe();
